# Why Project Atlas?

Project Atlas began as my CS-499 Computer Science Capstone, where I refactored an inventory management Android application using Kotlin, MVVM architecture, Room database, DAO layers, repository pattern, reactive data flows, optimized filtering logic, and secure coding practices.

After completing the capstone, I wanted to continue developing the project beyond an academic assignment.

My professional background includes warehouse operations, inventory control, supply chain processes, and data analysis. I have worked with operational systems, inventory movement, purchase orders, reporting, and process improvement. Because of that, I wanted to build a project that connects my real-world operations experience with my technical skills.

Atlas is designed to evolve from a basic inventory application into an enterprise-style supply chain intelligence platform.

The purpose of Atlas is to show that I can:

- Design relational databases
- Model supply chain data
- Write analytical SQL
- Build Python ETL workflows
- Validate and clean operational data
- Create Power BI dashboards
- Document technical decisions
- Deploy data systems to the cloud
- Communicate business value through analytics

Atlas is not just a portfolio project. It is a long-term professional case study that demonstrates continuous growth across software engineering, data analysis, business intelligence, and data engineering.
